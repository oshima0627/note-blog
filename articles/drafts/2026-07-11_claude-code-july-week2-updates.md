# Claude Code 7月第2週の更新：/doctor強化と自動プッシュ、安全対策

*2026年7月11日時点の情報です。料金・仕様は変わりやすいため、最新は公式の変更履歴でご確認ください。*

## この記事でわかること

- Claude Code の 7月6日〜9日（v2.1.202〜2.1.206）で入った主な変更
- `/doctor` の診断強化や `/commit-push-pr` の自動プッシュなど、日々の作業に効く実務ポイント
- Auto モードの「トランスクリプト改ざんブロック」が意味する安全面の変化

先日、Chrome版のGAや権限モードのManual化を含む「7月更新まとめ」をお届けしました。その続報として、直近数日で積み上がった細かくも実用的な変更を、公式の変更履歴（changelog）をもとに整理します。

## `/doctor` が本格的なセットアップ診断に

まず目立つのが `/doctor` の強化です。

**概要**：v2.1.203（7月7日）で、`/doctor` は「問題を診断して修正できる、フルのセットアップ点検」になりました。`/checkup` はそのエイリアス（別名）として使えます。さらに v2.1.206（7月9日）では、`/doctor` にチェック項目が追加されました。コードベースからClaudeが推測できる内容を削り、リポジトリに置いた `CLAUDE.md` の贅肉をそぎ落とす提案を出す、というものです。

**読者への影響**：環境設定でつまずいたときに、まず `/doctor` を叩けば原因の切り分けと修正案が得られます。`CLAUDE.md` が肥大化しがちなプロジェクトでは、点検で整理のきっかけをもらえます。筆者の見方ですが、設定トラブルの初動を1コマンドに集約できるのは、地味ながら効きます。

## `/commit-push-pr` と `/cd` の使い勝手が向上

**概要**：v2.1.206（7月9日）で、`/commit-push-pr` が `git push` を自動的に許可する範囲を広げました。従来の `origin` に加え、リポジトリで設定された push 先（`remote.pushDefault`、または単一リモートしかない場合はそのリモート）へのプッシュも自動で許可されます。同じバージョンで、`/cd` にディレクトリのパス補完（候補提示）が追加され、`/add-dir` と同じ挙動になりました。

**読者への影響**：`origin` 以外を主なプッシュ先にしているリポジトリでも、コミットからPR作成までの流れが止まりにくくなります。ディレクトリ移動もパス補完で速くなります。日常のちょっとした引っかかりを減らす改善です。

## Auto モードの安全対策：トランスクリプト改ざんをブロック

**概要**：v2.1.205（7月8日）で、Auto モードにセッションのトランスクリプト（会話ログ）ファイルの改ざんを防ぐルールが追加されました。

**読者への影響**：自律的に動くAutoモードでは、ログに偽の「承認しました」が紛れ込むと、本来は人間の確認が必要な操作が勝手に進むリスクがあります。今回のルールはその入口をふさぐものと理解できます。筆者の見方として、エージェントの自動実行を任せるほど、こうしたログ保護の重要性は増していきます。バックグラウンドや無人実行を使う人ほど押さえておきたい変更です。

## ワークフロー機能とリモート実行まわりの調整

**概要**：v2.1.202（7月6日）では、`/config` に「Dynamic workflow size」設定が加わりました。動的に作るワークフローの規模（エージェント数を小・中・大で）を調整する目安で、上限を強制するものではないとされています。あわせて `/workflows` のエージェント一覧レイアウトが見やすく整えられ、ワークフロー生成エージェントのテレメトリに実行IDなどの属性が追加されました。

また v2.1.204（7月8日）では、ヘッドレス（画面なし）セッションでの `SessionStart` フックのイベントが正しく流れず、リモートワーカーがフック実行中にアイドル扱いで停止されうる不具合が修正されました。

**読者への影響**：複数エージェントを束ねるワークフローを使う人は、規模の目安を設定で調整できます。CI やリモート環境で Claude Code を動かしている場合は、フック実行の安定性が上がります。

## まとめ

今回の更新は派手さこそありませんが、`/doctor` による診断、`/commit-push-pr` の自動プッシュ、Auto モードのログ保護と、「毎日の作業の摩擦を減らす」方向でそろっています。バージョンは短い間隔で上がるため、気づいたら `/doctor` で自分の環境を点検してみるのがおすすめです。

なお、変更履歴は日々更新されます。ここで触れた内容も、最新版では表現や挙動が変わる可能性があります。正確な仕様は必ず公式の changelog をご確認ください。
