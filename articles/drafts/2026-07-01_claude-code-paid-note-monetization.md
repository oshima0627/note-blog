# Claude Codeで書いた『有料note』で月10万円：AI×一次情報で作る売れる有料記事の設計図

実績もフォロワーもゼロ。それでも有料noteは売れます。武器は「あなたの一次情報」です。実際に使ったプロンプト、かかった工数、受け取った単価、そして失敗談です。

単品1,000円が月30〜50本、さらに月額500円の定期購読が100〜200人まで積み上がれば、手取りで月8万〜15万円が見えてきます。ただし初月から3ヶ月は月0〜1万円が現実で、月10万円は「継続」を前提とした到達目安です。

AIで量産された記事が飽和した今こそ、生の一次情報が効きます。それをClaude Codeで高速に売れる有料記事へ変換し、単品からマガジンへとストック収益化する設計図を、手数料・値付け・規約の公開事実つきで示します。誇大な断言はしません。

## この記事の対象読者と、到達できる収益の目安（先に結論）

この記事を最後まで読むと、次の3つが分かります。

- 一次情報を「売れるネタ」に変える手順
- 無料と有料の境界の引き方と、手取りベースの値付け
- フォロワー0からの集客導線とストック収益化

この記事の対象読者は、Claude や Claude Code を実務で使った経験がある人です。Claude Code とは、Anthropic社のAIモデルClaudeをターミナル上で使える開発支援ツールを指します。あなたが持つべき武器は「一次情報」です。一次情報とは、実際に使ったプロンプト、かかった工数、受け取った単価、そして失敗談など、あなた自身が体験した生の情報を指します。フォロワーは0人でも構いません。ネタは実績ではなく、あなたの手の中にあります。

到達できる収益の目安を先に示します。誇張しない現実的なレンジです。

- 有料単品1,000円 × 月30〜50本 = 月3万〜5万円
- 月額500円の定期購読 × 100〜200人 = 月5万〜10万円
- 両方を積み上げた合計 = 月8万〜15万円

ここで正直に書きます。初月から3ヶ月は月0〜1万円が普通です。最初の数本はほとんど売れません。月10万円という数字は、無料記事で集客し、有料記事を継続的に出し、定期購読を積み上げた先にある「継続前提の目安」です。1本出して翌月に10万円が入る話ではありません。

初期コストはほぼゼロです。固定費として実質唯一かかるのが Claude の利用料です。サーバー代も在庫も不要です。パソコンとClaudeへのアクセスがあれば今日から始められます。回収は早ければ数週間、遅くても数ヶ月で利用料は取り返せます。

最後に釘を刺します。「誰でも確実に稼げる」という断言はしません。稼げるかどうかは、あなたの一次情報の濃さと継続にかかっています。この記事はその成功確率を上げるための設計図です。保証ではありません。

## なぜ今『Claude Code実務者の有料note』が売れるのか（市場と差別化）

理由はシンプルです。AIで量産された一般論の記事が飽和し、価値が急落しているからです。「Claudeの使い方10選」のような記事は無料で無数に読めます。誰でも書ける情報にお金は払われません。相対的に、AIには書けない情報の価値が上がっています。

その情報こそ一次情報です。noteの「おすすめ」表示は、一次情報・実体験・失敗談を優遇する傾向があります。読者も運営も、生の体験を求めています。ここに大きな空白があります。

まだ埋まっていない切り口は、Claude / Claude Code 実務者の生の一次情報です。具体的には次のような情報です。

- 実際に業務で使ったプロンプトの全文
- 使い込んだ CLAUDE.md（プロジェクト固有の指示ファイル）の中身
- 権限設定やツール設定の具体的な構成
- 工数や単価などの数値データ
- ハマった失敗と、その解決方法

さらにClaude Codeには、ChatGPT系のツールと差別化できる固有の領域があります。サブエージェント（役割分担した複数AIの連携）、MCP（外部ツールと接続する仕組み）、権限設計などです。これらの実務ノウハウは日本語の情報がまだ少なく、需要に対して供給が追いついていません。ここを書ける人は限られます。

もう一つ強力な武器があります。メタな再現性です。この有料note自体を「Claude Codeでどう書いたか」という制作プロセスそのものが売り物になります。読者は記事の中身だけでなく、あなたの制作方法もまねしたいのです。作り方を見せることで、記事の説得力と価値が二重に上がります。

あなたの一次情報になりうるものを、チェックリストで確認してください。

- 業務や個人開発で実際に使ったプロンプトがある
- Claude Codeで工数を短縮した具体的な時間の数値がある
- 受注や納品で受け取った単価・報酬の数値がある
- サブエージェントやMCPなど固有機能を試した経験がある
- うまくいかず試行錯誤した失敗談と解決策がある
- 自分なりの設定ファイルやワークフローを持っている

3つ以上当てはまれば、あなたは売れる有料noteの原石を持っています。

## 収益の全体像：無料・単品・定期購読の『3層モデル』

有料noteで月10万円を目指すには、記事を単発で売る発想を捨てます。3つの層を組み合わせる「3層モデル」で設計します。

- 第1層：無料記事 = 集客資産。誰でも読める記事で発見してもらい、信頼の入口を作ります。
- 第2層：有料単品 = フロー収益。1本500〜1,980円で売る記事です。売った瞬間に収益が発生します。
- 第3層：定期購読マガジン / メンバーシップ = ストック収益。月額300〜1,000円で継続課金してもらう仕組みです。

核心となる動線はこうです。読者はまず無料記事であなたを発見します。次に有料単品で「この人の情報は金を払う価値がある」と信頼します。そして続編を束ねたマガジンで月額会員になります。無料で発見、単品で信頼、月額で継続。この流れを作ります。

ここでフローとストックの違いを理解してください。フロー収益は売るたびに発生しますが、売るのを止めれば収益も止まります。ストック収益は一度会員になれば毎月自動で入ります。月10万円を安定させるには、ストックが不可欠です。単品だけで月10万円は毎月50〜100本売り続ける必要があり、現実的ではありません。

収益シミュレーションを箇条書きで示します。まず単品のみの場合です。

- 単品1,000円 × 月30本 = 売上3万円 → 手取り約2.4万〜2.5万円
- 単品1,000円 × 月50本 = 売上5万円 → 手取り約4万〜4.2万円

次に単品と月額を組み合わせた場合です。

- 単品1,000円 × 月40本 = 売上4万円 → 手取り約3.2万〜3.4万円
- 月額500円 × 150人 = 売上7.5万円 → 手取り約6万〜6.3万円
- 合計手取り = 約9.2万〜9.7万円

手取りは手数料を引いた後の概算です。手数料の詳細はステップ3で説明します。数字を見れば明らかです。月10万円の本命はストック、つまり定期購読の積み上げです。

## ステップ1：一次情報の棚卸しと『売れるネタ』への変換

最初のステップは、あなたの中に眠る一次情報の棚卸しです。頭の中にある体験を、売れるネタに変換します。

棚卸しの観点は3つです。

- 実プロンプト・CLAUDE.md・設定：実際に使って効果があったもの
- 工数と単価の数値：短縮できた時間、受け取った報酬、時給換算
- 失敗と解決：ハマった箇所と、どう抜け出したか

金を払ってもらえるネタには条件があります。「再現できる具体性」と「他では読めない生の数値・失敗談」の両方を満たすことです。抽象的な心構えは無料で十分です。読者は「その手順をまねすれば自分も同じ結果を出せる」と思えるものに課金します。

ネタ化のフレームは4つの穴を埋めるだけです。「誰の」「どんな課題を」「どの手順で」「どんな結果（数値）で」解決したか。この4点が揃えば、それは売れるネタです。逆に、この4点が埋まらないネタは、まだ有料には早いネタです。

守秘には必ず注意してください。NDA（秘密保持契約）、企業の機密情報、個人情報は絶対にそのまま書きません。会社名は「あるSaaS企業」のように匿名化し、金額は「月20万〜30万円」のように数値レンジ化します。守秘を守りつつ再現性を伝えるのがプロの書き方です。

棚卸しに使えるテンプレートをコードブロックで示します。空欄を埋めてください。

```
【一次情報 棚卸しテンプレート】

■ 使ったプロンプト / 設定
- プロンプト内容：__________
- どのツール・機能を使ったか：__________（例：サブエージェント、MCP）

■ 数値データ
- 短縮できた工数：作業前 ___時間 → 作業後 ___時間
- 単価・報酬：__________ 円（レンジ化可）
- 時給換算：__________ 円

■ 課題と結果
- 誰の課題か：__________
- どんな課題か：__________
- どんな結果が出たか（数値）：__________

■ 失敗と解決
- ハマった箇所：__________
- 原因：__________
- 解決方法：__________

■ 守秘チェック
- 匿名化すべき固有名詞：__________
- レンジ化すべき数値：__________
```

棚卸しメモができたら、ネタ化をClaude Codeに手伝ってもらいます。次のプロンプトを使ってください。

```
以下は私がClaude Codeを実務で使った一次情報の棚卸しメモです。
このメモをもとに、有料noteとして売れるネタ案を5つ提案してください。

各ネタ案には以下を含めてください。
- 想定読者（誰の、どんな課題か）
- 提供する具体的な手順や情報
- 他の記事にはない差別化ポイント（生の数値・失敗談）
- 想定価格（500〜1,980円のレンジで）

【棚卸しメモ】
（ここに棚卸しテンプレートの内容を貼る）

注意：一般論ではなく、私の一次情報が主役になるネタにしてください。
```

このプロンプトでネタの方向性が定まります。ただし最終判断は自分でします。一番濃い一次情報を持つネタを1本選びます。

## ステップ2：Claude Codeで構成・下書きを高速執筆する（核心は自分で注入）

ネタが決まったら執筆です。ここでClaude Codeの役割を正しく理解してください。Claude Codeがやるのは、構成案づくり、下書き、言い換え、見出し最適化の「高速化」です。文章を速く形にするための道具です。

一方、記事の核心は必ず人間が注入します。核心とは、実プロンプトの全文、工数や単価の数値、失敗談、実績のスクリーンショットです。これらはあなたしか持っていません。AIには書けません。ここを自分で入れるからこそ、記事に金を払う価値が生まれます。AIに全部任せた記事は、AI量産記事と同じ運命をたどります。

note記法の制約も指示に組み込みます。noteはMarkdownの表（パイプとハイフンで作る表）に対応していません。表を書くと生のテキストのまま崩れて表示されます。だから比較や一覧は必ず箇条書きで書かせます。

AIが書いた下書きは必ず自分でファクトチェックします。AIは事実を間違えることがあります。手数料の数字、機能の仕様、価格の上限などは、あなたの目で公式情報と照合してください。数字を間違えた記事は信用を失います。

さらに制作プロセスを記事内で公開すれば、メタな再現性という付加価値がつきます。「この記事はClaude Codeでこう作りました」と見せるのです。読者はあなたのワークフローごと買いたくなります。

執筆はプロンプトを3段に分けると精度が上がります。棚卸しメモからアウトライン、アウトラインから下書きの順です。

```
【第1段：アウトライン作成】
以下の棚卸しメモをもとに、有料noteのアウトライン（見出し構成）を作ってください。
無料で読める導入と、有料にすべき核心部分を分けて示してください。
本文は5,000文字以上を想定してください。

【棚卸しメモ】
（ここに貼る）

【第2段：見出しごとの下書き】
上のアウトラインの各見出しについて、下書きを書いてください。
ただし「実プロンプト」「工数の数値」「単価」「失敗談」は
［★ここは著者が後で埋める★］と空欄にしてください。
その部分は私が一次情報を注入します。

【第3段：仕上げ】
下書き全体を、です・ます調で、一文60文字以内に整えてください。
比較や一覧は箇条書きで書いてください。表は使わないでください。
```

空欄にした★の部分に、あなたが一次情報を注入します。これで「AIの速度」と「人間の一次情報」が合体します。

CLAUDE.md風の「note執筆ルール」を設定ファイルにしておくと、毎回同じ品質で書けます。CLAUDE.md はプロジェクトの共通指示を書くファイルです。次のように設定します。

```markdown
# note執筆ルール（CLAUDE.md）

## 文体
- です・ます調で言い切る
- 一文は60文字以内
- 専門用語は初出で簡潔に説明する

## note記法の制約
- Markdownの表（縦棒で区切る表）は絶対に使わない
- 比較・一覧・価格表はすべて箇条書き（-）で書く

## 一次情報の扱い（最重要）
- 実プロンプト全文、工数・単価の数値、失敗談は
  ［★著者が埋める★］として必ず空欄にする
- これらをAIが創作してはいけない

## 誇大表現の禁止
- 「絶対」「誰でも確実に」は使わない
- 金額はすべて「目安」と明記する
```

この設定を置いておけば、Claude Codeが毎回ルールを守って下書きを出します。執筆の再現性が跳ね上がります。

## ステップ3：無料と有料の境界線を引き、手取りベースで値付けする

記事ができたら、無料と有料の境界線を引きます。これが売上を左右する重要な設計です。

境界の基本方針はこうです。導入、全体像、結論の一部までを無料で読ませます。そして具体的な手順、プロンプトの全文、設定ファイル、実績のスクリーンショットを有料にします。読者が「続きを読めば自分もできる」と確信した瞬間に有料ラインを置くのがコツです。無料部分で価値を証明し、有料部分で答えを渡します。

値付けは「手取りベース」で考えます。売上そのままが手元に残るわけではないからです。ここで手数料を公開事実として整理します。2026年時点の情報です。最新は必ずnote公式で確認してください。

プラットフォーム利用料は次の通りです。

- 有料記事・有料マガジン・メンバーシップ：売上の10%
- 定期購読マガジン：売上の20%

これに決済手数料が加わります。

- クレジットカード決済：5%
- PayPay決済：7%
- キャリア決済：15%

さらに売上を振り込む際、1回あたり270円の振込手数料がかかります。

これらを合計すると、手取りはおおむね売上の75〜85%になります。決済方法の内訳で変動します。だから値付けは手取りから逆算します。

手取り計算の具体例を箇条書きで示します。クレジットカード決済（利用料10%＋決済5%＝約15%が引かれる想定）で計算します。

- 1,000円の有料記事：手取り約850円
- 500円の有料記事：手取り約425円
- 月額500円のメンバーシップ100人：売上5万円 → 手取り約4.25万円
- 月額500円の定期購読マガジン100人（利用料20%）：売上5万円 → 手取り約3.75万円

同じ月額500円でも、メンバーシップと定期購読マガジンでは利用料が10%違います。手取りに大きく響きます。使い分けはステップ6で詳しく説明します。

価格帯の目安です。単品は500〜1,980円が売れやすいゾーンです。noteの価格設定は通常100円から50,000円まで可能です。プレミアム会員やnote proなら上限は10万円まで上がります。ただし高額にすればよいわけではありません。一次情報の濃さに見合った価格が正解です。

値付けチェックリストで確認してください。

- 手取りベースで価格を決めたか
- 無料部分だけで「価値がある」と伝わるか
- 有料部分に他では読めない一次情報が入っているか
- 価格が一次情報の濃さに見合っているか
- 手数料と振込手数料を計算に入れたか

## ステップ4：公開とnote内露出を最大化する（検索・ハッシュタグ・コンテスト）

記事を公開したら、note内での露出を最大化します。フォロワーが0人でも、note内の発見経路を使えば読まれます。

note内の主な発見経路は3つです。

- おすすめ：運営や仕組みが読者に推薦する枠
- 検索：キーワードで探す読者に届く経路
- 公式コンテスト：お題に応募して露出を得る経路

検索とハッシュタグの設計が第一歩です。読者が実際に打ち込む言葉を選びます。「Claude Code」「有料note」に加えて、記事の具体テーマ語を入れます。たとえば「サブエージェント」「受託 単価」「工数短縮」などです。抽象語より具体語のほうが、濃い読者に届きます。

一次情報や失敗談は、おすすめ表示で優遇を得やすい傾向があります。だから記事の冒頭で一次情報性を明示します。「実際に使ったプロンプトを全文公開します」「月◯万円の受注で使った設定を載せます」と最初に宣言するのです。読者にも運営にも、これは生の情報だと伝わります。

公式コンテストへの応募も有効です。AIやテクノロジーをお題にしたコンテストが定期的に開催されます。応募するとコンテスト専用の一覧に載り、リーチが底上げされます。応募は無料です。使わない手はありません。

公開前のチェックリストを箇条書きで示します。公開ボタンを押す前に必ず確認してください。

- 検索キーワードをタイトルと本文に入れたか
- ハッシュタグを設定したか（Claude・有料note・具体テーマ語）
- 導入で一次情報性を明示したか
- 開催中の公式コンテストに応募できるか確認したか
- 表（縦棒で区切る表）を使っていないか（崩れて表示されるため）
- 無料部分だけで価値が伝わるか読み返したか

## ステップ5：X・Zenn・はてブから集客導線をつくる（フォロワー0対策）

note内だけでは、フォロワー0からの立ち上がりは遅いです。外部チャネルから集客導線を作ります。ここがフォロワー0対策の核心です。

主な外部チャネルは3つです。それぞれ向き不向きがあります。

- X（旧Twitter）：拡散力が高く、断片的な一次情報の発信に向く
- Zenn：エンジニアが集まる技術記事プラットフォーム。技術者への信頼構築に向く
- はてなブックマーク：質の高い記事が拡散されやすく、まとまった知見に向く

Xの使い方が最重要です。一次情報の断片を無料で出します。数値や失敗談を1ツイートで見せ、詳細をnoteへ誘導するのです。「工数が5時間から30分になった話」「この設定でハマった失敗」など、続きが気になる断片を出します。全部は書きません。答えは有料noteに置きます。

はてなブックマークは拡散装置です。まとまった知見の記事がブックマークを集めると、一気に拡散されます。無料記事で狙うと効果的です。

内部の導線も設計します。無料記事から有料単品へ、有料単品から定期購読やメンバーシップへ。各記事の末尾に「次の一手」となるリンクを固定で置きます。読み終えた読者を次のステップへ必ず導きます。導線がないと、せっかくの読者が離脱します。

## ステップ6：単品をマガジン化してストック収益（月10万の本命）

ここが月10万円の本命です。単品をマガジン化して、ストック収益に変えます。

やり方はこうです。続編や関連する単品記事を束ね、マガジンにします。そして月額でまとめ買いできるようにします。読者は1本ずつ買うより、月額でまとめて読むほうがお得だと感じます。あなたは継続収益を得ます。両者に利点があります。

マガジンの選択肢は2つです。使い分けを箇条書きで整理します。

- 定期購読マガジン
  - 手数料：売上の20%
  - 条件：noteプレミアム（月500円）への加入が必要
  - 条件：運営の審査がある
  - 条件：月1回以上の更新義務がある
- メンバーシップ
  - 手数料：売上の10%
  - 条件：更新義務なし
  - 特徴：掲示板など会員コミュニティ機能を持てる

手取りだけ見ればメンバーシップが有利です。手数料が10%低く、更新義務もありません。無理なく続けたい人にはメンバーシップが向きます。一方、定期購読マガジンは「毎月必ず更新される安心感」を読者に与えられます。更新を続ける自信があるなら選択肢になります。最新の条件はnote公式で確認してください。

ストックが積み上がる仕組みを理解してください。マガジンには過去記事がすべて入っています。新規読者は、あなたを発見した時点で過去記事もまとめ買いできます。記事が増えるほどマガジンの価値が上がり、新規会員が入りやすくなります。これがストックの複利です。

ストック像を数字で示します。月額500円 × 100〜200人 = 月5万〜10万円です。これが安定して積み上がれば、単品と合わせて月10万円に届きます。ただし継続前提です。会員は価値を感じなければ解約します。毎月、新しい一次情報を届け続ける必要があります。

ネタ切れが最大の敵です。対策は一次情報の継続棚卸しです。実務を続ければ、新しいプロンプト、新しい失敗、新しい数値が毎月生まれます。それをステップ1のテンプレに書き溜めます。仕事そのものがネタの供給源になります。

単品からマガジン化する手順です。

1. 関連テーマの単品を3〜5本ためる
2. それらを束ねるマガジンを作成する
3. マガジンのコンセプトを一文で決める（例：Claude Code実務者の生ログ集）
4. 月額を設定する（300〜1,000円の範囲で）
5. 既存の単品読者に、マガジンのほうがお得だと案内する

月次の運用ルーティンも決めておきます。

- 月初：今月出す有料記事のネタを棚卸しメモから選ぶ
- 月中：Claude Codeで下書きを作り、一次情報を注入して公開する
- 随時：Xで一次情報の断片を投稿し、noteへ誘導する
- 月末：会員数と売上を記録し、反応が良かったテーマを次月に増やす

このルーティンを回すだけで、ストックは着実に積み上がります。

## 失敗しないための注意点：手数料・規約・誇大表現・守秘

最後に、失敗しないための注意点をまとめます。ここを外すと信用も収益も失います。

まず手数料と価格上限を公開事実として再掲します。2026年時点の情報です。

- 手取りは売上の概ね75〜85%（利用料＋決済手数料を引いた後）
- 価格の上限は通常50,000円、プレミアムやnote proで100,000円
- 振込手数料は1回あたり270円

これらは変わることがあります。最新はnote公式で必ず確認してください。

次にAI生成物の扱いです。noteの規約は2025年に改定され、AI生成物への向き合い方が問われるようになりました。だからこそ、一次情報と事実確認を記事の核に据えます。AIに全部書かせた薄い記事はリスクです。あなたの体験と、自分で確認した事実で記事を組み立ててください。規約の最新版はnote公式で確認してください。

誇大表現は禁止です。読者の信頼を一発で失います。やってはいけない表現と、言い換えの例を示します。

- NG：「誰でも確実に月10万円稼げます」
  - OK：「継続すれば月10万円が見えてきます（目安です）」
- NG：「絶対に売れる方法」
  - OK：「売れる確率を上げる方法」
- NG：「1本書けば不労所得」
  - OK：「続編を束ねればストック収益になります」

金額はすべて「目安」と明記します。断言は避けます。

返金と信用のリスクも直視してください。中身が薄いと読者は落胆し、返金を求めたり、悪い評判を広めたりします。一度失った信用は取り戻せません。価格に見合う一次情報を必ず入れてください。

守秘は繰り返し強調します。NDA、企業の機密情報、個人情報は絶対にそのまま載せません。会社名は匿名化し、金額はレンジ化します。守秘を守ることが、長く発信を続ける前提です。

公開前のセルフチェックリストです。

- 手数料を計算に入れて値付けしたか
- 誇大表現（絶対・誰でも確実に）を使っていないか
- 金額を「目安」と明記したか
- 価格に見合う一次情報を有料部分に入れたか
- NDA・機密・個人情報を匿名化・レンジ化したか
- 表を使わず箇条書きにしたか
