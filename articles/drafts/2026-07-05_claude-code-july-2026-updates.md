# Claude Code 7月更新まとめ：Chrome版GAとPR自動作成、既定権限がManualに

*※2026年7月5日時点の情報です。料金・仕様は変わりやすいため、最新は公式のCHANGELOG・リリースノートでご確認ください。*

## この記事でわかること
- Claude Codeがこの1週間で受けた主要アップデート（2026年6月30日〜7月4日）
- バックグラウンドエージェントが既定になり、完了時にドラフトPRまで作るようになった変化
- 既定の権限モードが「Manual」に変わった点と、日々の使い勝手への影響

Claude Code（ターミナルやIDEで動くAnthropic公式のコーディング支援ツール）は、2026年7月に入って立て続けにバージョンが更新されました。ここでは公式のリリースノートとCHANGELOGで確認できた変更のうち、日常的にClaude Codeを使うエンジニア・個人開発者に影響が大きいものを整理します。

## Claude in Chrome が一般提供（GA）に

**概要**：2026年7月2日のリリース（v2.1.198）で、「Claude in Chromeが一般提供（generally available）になりました」と公式CHANGELOGに記載されました。これまで限定的に提供されていたChrome上でClaudeを使う機能が、正式版として広く使えるようになった、という位置づけです。

**読者への影響**：ブラウザ上での作業とClaude Codeの連携を試したい人にとって、正式提供は導入の判断材料になります。ただし具体的な対応範囲や権限の扱いは変わりやすいため、実際に使う前に公式ドキュメントで前提条件を確認するのが安全です。

## バックグラウンド実行が既定に＋完了時にドラフトPRを作成

**概要**：同じくv2.1.198で、サブエージェントが「既定でバックグラウンド実行される」ようになりました。CHANGELOGには「サブエージェントは既定でバックグラウンド実行され、その間もClaudeは作業を続ける」とあります。あわせて、`claude agents` から起動したバックグラウンドエージェントは、コード作業が終わると「コミット・プッシュし、ドラフトPRを開く」挙動に変わりました。以前のように途中で止まって確認を求めるのではなく、成果物をドラフトPRの形でまとめてくれる、という流れです。

**読者への影響**：長めのタスクを投げて他の作業を進める、という使い方がしやすくなります。一方で、自動でコミット・プッシュ・PR作成まで進むため、対象ブランチやコミット内容を後から必ず確認する運用が前提になります。意図しない変更がそのままPRに乗る可能性を踏まえ、レビューの手順を決めておくと安心です。

## 通知フックと /dataviz スキルの追加

**概要**：v2.1.198では、`claude agents` に「入力待ち」または「完了」時にフック（`agent_needs_input` / `agent_completed`）を発火させる通知の仕組みも加わりました。加えて、チャートやダッシュボードの設計指針を扱う `/dataviz` スキルが追加されています。

**読者への影響**：バックグラウンドで走らせたエージェントの状態を、フック経由で自分の通知手段（デスクトップ通知など）につなげられます。放置しがちなバックグラウンド作業の「終わった／止まった」を拾いやすくなるのは実務的なメリットです。

## 既定の権限モードが「Manual」に変更

**概要**：2026年7月3日のv2.1.200で、既定の権限モードが「default」から「Manual」に変更されました。CHANGELOGによれば、この変更はCLI・VS Code・JetBrains拡張にわたって適用されます。あわせて、`AskUserQuestion` のダイアログが自動で進まなくなり、必要ならidleタイムアウトを `/config` で有効化できる、とされています。

**読者への影響**：初期状態でより明示的に確認を求める挙動になった、と理解しておくとよいでしょう。自動化を重視して以前の挙動に近づけたい場合は、権限モードや `/config` の設定を見直す必要があります。既定値が変わったことに気づかず「急に確認が増えた」と感じるケースを避けるため、アップデート後は自分の設定を一度確認することをおすすめします。

## 背景：Sonnet 5が既定モデルに（既報）

これらのツール更新の少し前、2026年6月30日にClaude Sonnet 5が発表され、7月1日のv2.1.197でClaude Codeの既定モデルになりました。ネイティブで100万トークンのコンテキストに対応します。モデル面の詳細は別の無料記事で扱っているため、ここでは背景として触れるにとどめます。

## 筆者の見方

ここ1週間の更新は「バックグラウンドで走らせ、成果をPRで受け取る」という使い方へ寄せた印象です（筆者の見方）。同時に既定権限がManualになったのは、自動化と安全確認のバランスを取り直す動きと考えられます。自動化の恩恵を受けつつ事故を防ぐには、対象ブランチとPR内容を必ず人が確認する運用がこれまで以上に重要になりそうです。
