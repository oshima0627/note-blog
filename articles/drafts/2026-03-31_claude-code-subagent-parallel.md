# Claude Codeサブエージェント並列処理の全技術

**対象読者**: Claude Codeを使い始めた開発者、AIコーディングの効率を上げたい方
**この記事で得られること**: サブエージェントの仕組みと並列処理による開発高速化の具体的手法

---

## はじめに

Claude Codeで大規模なタスクに取り組むと、1つのエージェントがすべてを順番に処理します。ファイル調査、コード修正、テスト実行……。待ち時間が積み重なり、もどかしさを感じた経験はないでしょうか。

この課題を解決するのが「サブエージェント」です。タスクを分割し、複数のエージェントに並列で任せる仕組みです。本記事では、サブエージェントの基本から実践的な活用術までを解説します。

---

## サブエージェントとは何か

サブエージェントは、メインのClaude Codeから呼び出される独立したエージェントです。Agent toolを使って起動します。

重要な特徴は、**独立したコンテキストウィンドウ**で動作する点です。メインエージェントの文脈とは切り離されているため、メインのコンテキストが肥大化しません。処理結果だけを要約して返すため、効率的に情報を受け渡せます。

ただし、サブエージェントはさらにサブエージェントを生成できません。ネストは不可という制約があります。

---

## subagent_typeとカスタムエージェントの違い

サブエージェントには、Agent toolの `subagent_type` パラメータで指定する**組み込み型**と、自分で定義する**カスタムエージェント**の2種類があります。

### 組み込み型（subagent_type）

Agent toolの呼び出し時に `subagent_type` で指定します。値は文字列で、以下の3種類が使用可能です。

- **`"general-purpose"`**: 汎用型（デフォルト）。読み書きすべての操作が可能
- **`"Explore"`**: 読み取り専用の調査特化型。ファイル検索・コード検索・読み取りツールのみが使える。コードベースの分析に最適
- **`"Plan"`**: 設計計画の策定に特化。読み取りツールのみで書き込み不可。方針を整理してから着手したい場合に有用

`"Explore"` と `"Plan"` は大文字始まりである点に注意してください。

### ビルトインカスタムエージェント

Claude Codeに同梱されているカスタムエージェント定義です。`subagent_type` ではなく、エージェント名で呼び出します。

- **claude-code-guide**: Claude Codeのドキュメント参照用
- **statusline-setup**: ステータスライン設定用

この2つは組み込み型とは呼び出し方が異なるため、混同しないよう注意してください。

## 並列実行の仕組み

サブエージェントを並列で動かす方法はシンプルです。**1つのメッセージ内で複数のAgent tool callを同時に発行する**だけで、それらは並列実行されます。

たとえば、3つのモジュールの仕様を同時に調べたい場合、3つのExplore型サブエージェントを一度に呼び出します。順番に実行すると3倍の時間がかかる処理を、ほぼ同時に完了できます。

さらに `run_in_background: true` を指定すると、バックグラウンド実行が可能です。メインエージェントはサブエージェントの完了を待たずに自分の作業を続けられます。バックグラウンドのサブエージェントが完了すると、システムが自動的にメインエージェントに通知します。結果を能動的にポーリングする必要はありません。

## カスタムサブエージェントの定義と呼び出し

`.claude/agents/` ディレクトリにMarkdownファイルを置くと、独自のサブエージェントを定義できます。

### 定義例：テストランナー

```markdown
---
name: test-runner
description: テストスイートを実行して結果を報告
tools: Read, Bash, Glob
model: sonnet
---
テストを実行し、失敗したテストの原因を分析してください。
```

ポイントは `tools` でアクセス可能なツールを最小限に絞ることです。不要な権限を与えないことで、意図しない動作を防げます。また、YAMLフロントマターの `model` フィールドでサブエージェントが使用するモデルを個別に指定できます。

### 呼び出し方

CLAUDE.mdやプロンプト内で以下のように指示します。

```
test-runnerエージェントを使って、src/authモジュールのテストを実行してください。
```

Claude Codeは `.claude/agents/test-runner.md` を認識し、定義に従ってサブエージェントを起動します。組み込み型のように `subagent_type` で指定するのではなく、エージェント名で参照する点が異なります。

## worktree分離で安全に並列書き込み

並列実行で最も注意すべきは**ファイルの競合**です。複数のサブエージェントが同一ファイルに書き込むと、内容が壊れます。

この問題を解決するのが `isolation: "worktree"` です。サブエージェントごとに一時的なgit worktreeを作成し、作業ディレクトリを完全に分離します。

### worktree分離後の統合フロー

各サブエージェントの作業が完了すると、以下のように処理されます。

- **変更がなかった場合**: worktreeは自動的にクリーンアップされます
- **変更があった場合**: worktreeのパスとブランチ名が結果として返されます

重要な点として、**worktreeの変更はメインの作業ディレクトリに自動マージされません**。メインエージェントまたはユーザーが、返されたブランチを確認し、手動でマージする必要があります。

競合を避ける設計のコツは、**各サブエージェントが担当するファイル群を明確に分離する**ことです。モジュールA担当とモジュールB担当を分ければ、マージ時の競合リスクを最小限に抑えられます。

## 実践ユースケース：複数モジュールの同時リサーチ

概念だけでなく、具体的なプロンプト例を示します。大規模コードベースで認証・決済・通知の各モジュール仕様を同時に調査するケースです。

CLAUDE.mdに以下のように記述します。

```
以下の3つの調査を並列で実行してください（すべてExplore型）：

1. src/auth/ 配下の認証フローを分析し、認証方式・トークン管理・セッション処理を要約
2. src/payment/ 配下の決済処理を分析し、対応決済手段・エラーハンドリング・冪等性担保を要約
3. src/notification/ 配下の通知システムを分析し、通知チャネル・リトライ戦略・テンプレート管理を要約
```

3つのExplore型サブエージェントが並列起動し、それぞれ独立して担当モジュールを調査します。順次実行なら合計数分かかる調査が、並列実行で体感1/3の待ち時間に短縮されます。

その他の活用例としては、コードレビューの自動化（スタイルチェック・セキュリティスキャン・カバレッジ分析を同時実行）や、worktree分離を使った複数モジュールの同時リファクタリングがあります。

## コスト最適化のコツ

サブエージェントはそれぞれ独立したコンテキストウィンドウを消費します。多数を同時起動するとトークンコストが増大します。

`CLAUDE_CODE_SUBAGENT_MODEL` 環境変数でサブエージェントのデフォルトモデルを指定できます。

```bash
export CLAUDE_CODE_SUBAGENT_MODEL=sonnet
```

また、カスタムエージェント定義ファイルのYAMLフロントマターで `model: sonnet` のように個別指定することも可能です。

Claude APIの入力トークン単価はモデルによって大きく異なります。おおよその目安として、Opusに対してSonnetは約1/5、Haikuは約1/30程度のコストです（最新の正確な料金はAnthropicの公式料金ページで確認してください）。調査タスクや定型チェックならSonnetで十分な精度が出るため、メインにOpus・サブエージェントにSonnetという構成でコストを大幅に抑えられます。

また、サブエージェントの戻り値が大きいとメインのコンテキストを圧迫します。「情報収集をして、小さな要約を返す」という使い方が最もコスト効率に優れています。

---

## 注意すべき落とし穴

- **ネスト不可**: サブエージェントがさらにサブエージェントを呼ぶことはできません
- **ファイル競合**: worktree分離なしでの同一ファイル同時書き込みは厳禁です
- **自動マージなし**: worktree分離を使った場合でも、変更の統合は手動で行う必要があります
- **context rot**: コンテキストが肥大化すると精度が低下します。戻り値は簡潔に保ちましょう
- **権限の最小化**: `tools` 指定で必要なツールだけを許可してください

---

## まとめ

- **サブエージェントは独立コンテキストで動作し、並列実行で開発速度を大幅に向上させる**
- **subagent_typeの組み込み型とカスタムエージェントは呼び出し方が異なる。用途に応じて使い分ける**
- **worktree分離とファイル担当の明確化で、安全な並列書き込みが実現できる。ただし統合は手動で行う**
- **コスト最適化にはモデルの使い分けと戻り値の簡潔化が重要である**

サブエージェントを使いこなすことで、Claude Codeの生産性は飛躍的に高まります。まずはExplore型での並列リサーチから試してみてください。
