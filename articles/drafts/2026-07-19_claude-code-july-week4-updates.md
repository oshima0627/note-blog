# Claude Code 7月第4週の更新：権限チェックの抜け穴修正と暴走防止の上限

**2026年7月18日時点の情報です**（Claude Code v2.1.212〜2.1.214、7月17日〜18日リリース分）。料金・仕様は変わりやすいため、最新は公式ドキュメントでご確認ください。

## この記事でわかること

- 権限（パーミッション）チェックの抜け穴が複数まとめて修正されたこと
- WebSearchやサブエージェントの「暴走」を止めるセッション上限が追加されたこと
- `/fork`がバックグラウンドセッションのコピーに変わり、`/subtask`が新設されたこと

先週分（v2.1.207〜2.1.211）は別記事にまとめています。本記事はその続きにあたる、7月17日〜18日のリリース分です。

## 権限チェックの抜け穴が複数修正（v2.1.214）

7月18日のv2.1.214は、権限（パーミッション）チェックに関する修正が特に多いリリースです。権限チェックとは、ファイル編集やコマンド実行の前にClaude Codeが「実行してよいか」を判定・確認する仕組みで、安全に自動化するための要です。

公式changelogによると、主に次の修正が入りました。

- `Edit(src/**)`のような単一セグメントの許可ルールが、`<cwd>/dir`だけでなくツリー内のどこにある`dir/`への書き込みも自動承認してしまう問題を修正
- Windows PowerShell 5.1セッションでの権限チェック回避（バイパス）を修正
- ファイルディスクリプタのリダイレクト形式で権限チェックが「安全側に倒れる（fail closed）」よう修正
- 10,000文字を超える非常に長いコマンドを誤判定していた問題を修正（今後は常に確認を求める）
- `[[ ]]`比較内のzsh変数の添字・修飾子を無害なテキストとして扱っていた問題を修正
- 一部の`help`・`man`コマンドが危険なオプションを実行し得るのに自動承認されていた問題を修正

**読者への影響**: 許可ルールやバックグラウンド実行に依存して自動化している人ほど関係します。これまで意図せず自動承認されていたケースが確認を求めるようになるため、挙動が安全側に寄ります。単一セグメントの`dir/**`ルールを使っている場合、任意の深さにマッチさせるには`**/dir/**`と書く必要がある点にも注意してください。

出典: 公式changelog（v2.1.214）

## 「暴走」を止めるセッション上限（v2.1.212）

7月17日のv2.1.212では、ツールの実行回数に上限が設けられました。ループが止まらず延々とツールを呼び続ける「暴走」を防ぐための変更です。

- WebSearchツールの呼び出しにセッション全体の上限（既定200回、`CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION`で調整可）
- サブエージェント起動にセッションごとの上限（既定200、`CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`で変更可）。`/clear`で上限がリセットされます
- 2分を超えて実行中のMCPツール呼び出しは自動でバックグラウンドに移り、セッションを使い続けられる（`CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS`でしきい値変更・無効化が可能）

**読者への影響**: 大量のサブエージェントを並列で走らせたり、検索を多用する自動ワークフローを組んでいる人は、既定の上限に達する可能性があります。意図的に多く回す構成では、環境変数で上限を引き上げておくとよいでしょう。逆に、コスト暴走が心配な人にとっては安全装置になります。

出典: 公式changelog（v2.1.212）

## `/fork`がバックグラウンドセッションに、`/subtask`新設（v2.1.212）

同じくv2.1.212で、`/fork`コマンドの挙動が変わりました。これまで`/fork`は会話をその場のサブエージェントに引き継ぐ形でしたが、今後は会話を**新しいバックグラウンドセッション**（`claude agents`に別の行として並ぶ）へコピーし、その間も手元の作業を続けられるようになります。

従来のセッション内サブエージェントを起動する機能は、新コマンド`/subtask`に分離されました。

**読者への影響**: 「今の文脈を保ったまま別方向を試したい」ときに、元の作業を止めずに枝分かれできます。バックグラウンドで走るため、並行して複数の検討を進めやすくなります。用途に応じて`/fork`（別セッションへ複製）と`/subtask`（その場でサブタスク）を使い分けることになります。

出典: 公式changelog（v2.1.212）

## その他の注目点

- `/resume`をエージェントビューで入力すると、過去セッション（リストから削除したものを含む）のピッカーが開き、選んだものをバックグラウンドセッションとして再開できるようになりました（v2.1.212）
- `claude auto-mode reset`が追加され、自動モードの設定を既定に戻せます（確認プロンプトあり。`--yes`で省略）（v2.1.212）
- プラン（計画）モードが、権限プロンプトなしにファイルを変更するBashコマンドを自動実行してしまう問題が修正されました（v2.1.212）
- 高い虐待的・脱獄（ジェイルブレイク）的な利用に対し、Claudeがセッションを終了できる`EndConversation`ツールが追加されました（v2.1.214）
- 画像を多く含む会話が「Request too large」で失敗する問題や、WebSearch/fetchの信頼性（529エラーのリトライ）も改善されています（v2.1.212）

出典: 公式changelog（v2.1.212、v2.1.214）

## 筆者の見方

今回の目玉は、派手な新機能よりも「安全に自動化するための土台」の整備だと考えられます。権限チェックの抜け穴修正と、WebSearch・サブエージェントの上限は、いずれも自動実行・バックグラウンド運用を前提にした変更です。Claude Codeを人が張り付かずに動かす使い方が広がっていることの表れ、というのが筆者の見方です。

自動化を積極的に使っている人は、（1）自分の許可ルールが意図通りか、（2）上限に引っかからないか、の2点を一度確認しておくと安心です。なお個々の数値・挙動は変わりやすいため、最新は公式changelogでご確認ください。
