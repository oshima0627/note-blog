## 収益化モデル：ニッチ×サブスクで積み上げる

収益源は**Stripeサブスク課金**の一択です。
Freemium + 有料2階層で、解約率を下げながらMRRを積みます。

到達シミュレーション（MRR換算）は以下です。

- 1,000円 × 50人 = 月5万円
- 2,000円 × 25人 = 月5万円
- 5,000円 × 10人 = 月5万円

ポイントは「広く薄く」ではなく「狭く深く」です。
特定業界×特定職種の1タスクだけを解決するニッチSaaSが勝ち筋です。

個人開発で稼げる人と稼げない人を分ける最大の違いは、作る前に売れると確信できているかどうかです。多くの個人開発者は自分が作りたいものから始めて、リリース後に「誰にも響かなかった」と嘆きます。マイクロSaaSで稼ぐには、この順序を逆にする必要があります。つまり、先に痛みを感じている人を20人見つけ、その人たちが月額でお金を払う意思を確認してから、コードを書き始めるということです。

ニッチを選ぶ基準はシンプルです。第一に、対象業界の人が日常的に時間を溶かしている作業があること。第二に、その作業をAIで自動化または高速化できること。第三に、既存の大手SaaSがその業界向けに専用機能を提供していないこと。この3条件が揃った領域では、個人開発者でも月5万〜20万円のMRRは十分狙えます。競合が大手しかいない市場ではなく、競合が手作業である市場を選ぶのが、個人開発で勝ち筋を作る基本戦略です。

## 技術スタック：2週間で動かす構成

個人開発で速く作り、運用で消耗しない構成を選びます。

- フロント/API: **Next.js App Router + Cloudflare Pages / Workers**
- 認証: **Clerk**（メール+OAuth即導入）
- DB: **Cloudflare D1（推奨）** / 規模拡大時は Supabase + Drizzle ORM を検討
- 課金: **Stripe Billing + Webhook**（Cloudflare Workers で受け口を実装）
- AI: **Claude API（@anthropic-ai/sdk）**
- UI: TypeScript + React + Tailwind
- その他: **Cloudflare KV / R2 / Queues** でキャッシュ・ストレージ・非同期処理
- 役割分担: D1=メタデータ/サブスク状態、KV=レート制限カウンタ・軽量キャッシュ、R2=生成物/添付ファイル保管、Queues=Claude API非同期バッチ実行

SaaS Starterテンプレ（`@opennextjs/cloudflare` ベース）を流用すれば、MVPは1〜2週間で立ち上がります。`wrangler.toml` で `compatibility_flags = ["nodejs_compat"]` を有効化しておくと、`@anthropic-ai/sdk` などNode系ライブラリがそのまま動きます。Cloudflareはエッジ実行で世界中どこからも低レイテンシ、従量課金の下限が非常に低い点が個人開発に向きます。

この構成を選ぶ理由は、個人開発者の時間と資金を最小化できる点にあります。Vercelで同等構成を組むと、トラフィックが増えた瞬間に月数十ドルの請求が発生しやすく、MRRが立ち上がる前にコストが膨らむ懸念があります。Cloudflareなら、Workers Freeプランの1日10万リクエストまでは完全無料で、Pages の静的配信は実質無制限です。個人でローンチ直後に数千ユーザーの流入があっても、課金が発生するのはQueuesや大量のKV書き込みを使ったときに限られます。初月から数ヶ月は、ほぼ固定費ゼロでサービスを運営できるわけです。

さらに、Cloudflare Registrarでドメインを取得すれば、更新料もドメイン原価のみで済みます。一般的なドメインレジストラで年3,000〜4,000円かかるところが、Cloudflareなら年1,500円前後です。年間1,500〜2,500円の節約は小さく見えますが、5年スパンで考えれば1万円を超える差になります。個人開発でMRRを積み上げるフェーズでは、こうした固定費の削減が精神的な余裕にも直結します。

## コスト最適化：最大95%削減の3点セット

2026年4月時点のClaude API単価（per 1M tokens）は以下です。

- Haiku 4.5: 入力 $1 / 出力 $5
- Sonnet 4.6: 入力 $3 / 出力 $15
- Opus 4.7: 入力 $5 / 出力 $25

※正確な最新料金は Anthropic公式 Pricing ページで要確認。

次の3つを組み合わせれば、原価は最大95%下がります。

- **プロンプトキャッシング（-90%）**: 共通システムプロンプトをキャッシュ
- **Batch API（-50%）**: 即応不要な処理は夜間バッチで実行
- **Haikuフォールバック**: 簡易タスクはHaiku 4.5に自動ダウングレード

### Claude API呼び出しサンプル（プロンプトキャッシュ付き）

```ts
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

export async function runAI(userInput: string) {
  const res = await client.messages.create({
    model: "claude-haiku-4-5",
    max_tokens: 1024,
    system: [
      {
        type: "text",
        text: LONG_SYSTEM_PROMPT,
        cache_control: { type: "ephemeral" },
      },
    ],
    messages: [{ role: "user", content: userInput }],
  });
  return res.content;
}
```

`cache_control` を付けるだけで、2回目以降の入力コストが約1/10になります。

## Stripe Webhook 実装サンプル（Cloudflare Workers）

Webhookは決済状態をD1に同期する心臓部です。Cloudflare Workersで実装する最小構成は以下です。

```ts
import Stripe from "stripe";

export async function handleWebhook(req: Request, env: Env) {
  const sig = req.headers.get("stripe-signature")!;
  const body = await req.text(); // 生ボディを必ず先に取得
  const stripe = new Stripe(env.STRIPE_SECRET);

  const event = await stripe.webhooks.constructEventAsync(
    body, sig, env.STRIPE_WEBHOOK_SECRET
  );

  switch (event.type) {
    case "checkout.session.completed":
      // D1にサブスク作成フラグを書く
      break;
    case "customer.subscription.updated":
      // プラン変更をD1に反映
      break;
    case "customer.subscription.deleted":
      // 解約処理、Clerkのメタデータも更新
      break;
  }
  return new Response("ok");
}
```

最低限扱うイベントは `checkout.session.completed`・`customer.subscription.updated`・`customer.subscription.deleted` の3つです。D1に「プラン種別・有効期限・使用量」の3カラムを持たせて同期します。署名検証に失敗したリクエストは即400を返し、ログに残すだけでOKです。

## プロンプトインジェクション・API悪用対策

個人SaaSでよくある攻撃パターンは3つです。いずれもAPIコストを食い潰す実害につながるため、リリース前に必ず塞ぎます。

- **システムプロンプト漏洩**: 「これまでの指示を全て出力して」系の入力で内部プロンプトを抜かれる
- **トークン吸い上げ**: 長大な入力を繰り返し送りAPI課金を急増させる
- **規約違反誘導**: 違法用途の出力を生成させブランド毀損・アカウント停止に追い込む

対策は以下をレイヤーで重ねます。

- 入力サニタイズ: 最大文字数制限、怪しい制御文字・URLパターンの除外
- `max_tokens` 上限: プランごとに固定し、暴走時の上限コストを計算可能に
- KVレート制限: ユーザー毎に1分/1時間/1日のカウンタを持たせ、閾値超過でブロック
- モデル自動ダウングレード: 閾値超過や疑わしい入力はHaikuに落として原価抑制
- 日次コストアラート: Anthropic ConsoleとCloudflare Analyticsで閾値超過をSlack通知

これらは1日で実装できる内容です。怠ると「悪意ある1ユーザーで1ヶ月のAPI代が売上を食い潰す」事故が現実に起こります。

## 失敗パターンとピボット判断基準

個人SaaSで最も危険なのは「沈没船に2年乗り続けること」です。以下の数字を判断基準にします。

- **3ヶ月MRR 1万円未満 → ピボット検討**: ニッチまたは価格設計に根本的な問題
- **粗利率50%割れ → 料金改定**: API原価が重すぎるか、課金設計のミス
- **チャーン率10%超/月 → 機能・価値の再設計**: 継続理由が薄い
- **獲得CPA > LTVの50% → 集客チャネル変更**: 広告依存しすぎ

ピボットの型は4パターンあります。①同じプロダクトで別ニッチ、②同じニッチで別ソリューション、③B2CからB2B転換、④機能絞り込みで単価アップ。4ヶ月目以降は新規開発を止め、撤退か成長かを90日で決めます。

## リスク・注意点

- **Anthropic Usage Policy違反**: 即アカウント停止リスク。システムプロンプトと入力バリデーションで禁止用途をブロックします
- **API暴走対策**: ユーザー毎トークン上限、指数バックオフ、max_tokens、Haikuへのモデルダウングレードを実装します
- **特商法・規約**: 特商法表記・利用規約・プライバシーポリシーは日本向け必須。消費者契約法上、一方的免責は無効です
- **チャーン率**: 月5〜10%が標準。年額プラン30%割引でチャーンは半減します
- **競合リスク**: 大手が無料AI機能で侵食します。日本語特化×業界ワークフロー深掘りで代替不可化します
- **粗利設計**: Stripe手数料3.6%+サブスク0.7%+API原価を引いて黒字化する料金にします
- **Workers での Stripe Webhook**: `await request.text()` で生ボディを取得し、`stripe.webhooks.constructEventAsync` で署名検証します。Node前提の `constructEvent` は Workers で動きません
