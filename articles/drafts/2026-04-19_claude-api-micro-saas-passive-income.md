## 収益化モデル：ニッチ×サブスクで積み上げる

収益源は**Stripeサブスク課金**の一択です。
Freemium + 有料2階層で、解約率を下げながらMRRを積みます。

到達シミュレーション（MRR換算）は以下です。

- 1,000円 × 50人 = 月5万円
- 2,000円 × 25人 = 月5万円
- 5,000円 × 10人 = 月5万円

ポイントは「広く薄く」ではなく「狭く深く」です。
特定業界×特定職種の1タスクだけを解決するニッチSaaSが勝ち筋です。

### 粗利フォーミュラと試算

覚えるのはこの1行だけでOKです。

- **フォーミュラ**: `月額 × 0.93 − API原価 ≧ 粗利目標`

係数0.93はStripe手数料（3.6%+サブスク0.7%+α）を差し引いた概算です。

## コスト最適化：最大95%削減の3点セット

2026年4月時点のClaude API単価（per 1M tokens）は以下です。

- Haiku 4.5: 入力 $1 / 出力 $5
- Sonnet 4.6: 入力 $3 / 出力 $15
- Opus 4.7: 入力 $5 / 出力 $25

※正確な最新料金は Anthropic公式 Pricing ページで要確認。

次の3つを組み合わせれば、原価は最大95%下がります。

- **プロンプトキャッシング（-90%）**: 共通システムプロンプトをキャッシュ
- **Batch API（-50%）**: 即応不要な処理は夜間バッチで実行
- **Haikuフォールバック**: 簡易タスクはHaiku 4.5に自動ダウングレード

### Claude API呼び出しサンプル（プロンプトキャッシュ付き）

```ts
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

export async function runAI(userInput: string) {
  const res = await client.messages.create({
    model: "claude-haiku-4-5",
    max_tokens: 1024,
    system: [
      {
        type: "text",
        text: LONG_SYSTEM_PROMPT,
        cache_control: { type: "ephemeral" },
      },
    ],
    messages: [{ role: "user", content: userInput }],
  });
  return res.content;
}
```

`cache_control` を付けるだけで、2回目以降の入力コストが約1/10になります。
