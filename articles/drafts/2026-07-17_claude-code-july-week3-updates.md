# Claude Code 7月第3週の更新：フリーズ解消とメモリリーク修正

**2026年7月15日時点の情報です**（Claude Code v2.1.207〜2.1.211、7月11日〜15日リリース分）。料金・仕様は変わりやすいため、最新は公式ドキュメントでご確認ください。

## この記事でわかること

- 長い応答でターミナルが固まる問題が修正されたこと
- 長時間セッションのメモリリーク・重さが大きく改善されたこと
- Bedrock/Vertex/Foundryでの自動モード（Auto mode）や安全性の変更点

## ターミナルのフリーズ・入力遅延が解消

v2.1.207で、長いリスト・表・段落・コードブロックを含む応答をストリーミング中に、ターミナルが固まってキー入力が遅れる問題が修正されました。日常的に長い出力を扱う人ほど体感しやすい修正です。

あわせて、応答のストリーミング完了時に画面が回答の先頭より上へ飛んでしまう挙動も直っています。

**読者への影響**: 大きな差分やログを表示させる作業でのストレスが減ります。該当バージョン以降へ更新する価値があります。

出典: 公式changelog（v2.1.207）

## 長時間セッションのメモリリーク・重さを改善

v2.1.208は安定性・性能面の修正が特に多いリリースです。公式changelogによると、長時間セッションでの複数のメモリリークが修正されました。具体的には、MCPのstdioサーバのstderrがサーバごとに最大64MBまで蓄積する問題、LSPドキュメントが開いたままになる問題（50件のLRU上限に変更）などです。

性能面では、次のような改善が挙げられています。

- セッションのトランスクリプト（記録）サイズを、編集の多いセッションで最大79倍削減
- MCPツールが多い環境で、ツール実行のラウンドを最大7倍高速化
- 許可（deny/ask）ルールが多いセッションでの毎ターンの遅延を、ルールのマッチャをキャッシュして解消

**読者への影響**: MCPサーバやLSPを多数つないだまま長時間作業する構成ほど恩恵が大きい修正です。「使っているうちに重くなる」と感じていた場合は改善が期待できます。

出典: 公式changelog（v2.1.208）

## Auto mode（自動モード）がBedrock/Vertex/Foundryで既定に

v2.1.207で、Bedrock・Vertex AI・FoundryにおいてAuto mode（自動モード）が`CLAUDE_CODE_ENABLE_AUTO_MODE`のオプトインなしで利用できるようになりました。無効化したい場合は設定の`disableAutoMode`で切り替えます。

またv2.1.210では、Auto modeの権限分類器が外部セッションで既定的にSonnet 5を使うよう改善され、`/doctor`がこれらの環境でAuto modeの提案をスキップしていた不具合も修正されています。

**読者への影響**: クラウド経由（Bedrock/Vertex/Foundry）でClaude Codeを使うチームは、自動モードの前提が変わります。意図せず有効になっては困る場合は`disableAutoMode`の確認をおすすめします。

出典: 公式changelog（v2.1.207、v2.1.210）

## 安全性・プロンプトインジェクション対策の強化

危険な操作や外部入力への防御も複数入りました。公式changelogより主なものです。

- サブエージェントが読み込んだ内容を経由する間接的なプロンプトインジェクションに対し、Agentツールを強化（v2.1.210）
- `ultracode`キーワードのオプトインが、Webhookのペイロードや中継されたPRコメントなど、人間由来でない入力で発火しないよう修正（v2.1.210）
- `$(…)`やバッククォートを含むコマンド内の破壊的削除（例: `rm -rf ~`）が、`--dangerously-skip-permissions`や自動モードでも確認を求めるよう変更（v2.1.208）

**読者への影響**: 自動実行やバックグラウンド運用をしている人ほど関係します。外部コメントやWebhookを起点に動かす構成では、挙動が安全側に寄りました。

出典: 公式changelog（v2.1.208、v2.1.210）

## アクセシビリティと操作性

v2.1.208では入力・操作まわりの新機能も追加されました。

- スクリーンリーダー向けのプレーンテキスト表示モードを追加（`claude --ax-screen-reader`、環境変数`CLAUDE_AX_SCREEN_READER=1`、または設定`axScreenReader`）
- `vimInsertModeRemaps`設定を追加し、vimモードで`jj`のような2キー入力をEscにマップ可能に
- 全画面モードで、複数選択メニューや「Other」入力欄のマウスクリックに対応

**読者への影響**: キーボード中心・スクリーンリーダー利用など、環境に合わせた操作がしやすくなりました。

出典: 公式changelog（v2.1.208）

## 自動化・SDK利用者向け：サブエージェントのテキスト転送

v2.1.211で、`--forward-subagent-text`フラグと環境変数`CLAUDE_CODE_FORWARD_SUBAGENT_TEXT`が追加され、stream-json出力にサブエージェントのテキストや思考を含められるようになりました。

**読者への影響**: `claude -p`やSDKでマルチエージェントを組み、途中経過をログとして拾いたい人に有用です。

出典: 公式changelog（v2.1.211）
