# Claude Sonnet 5がClaude Codeの新既定モデルに：料金と使い分け

**この記事は2026年7月3日時点の情報です。** 料金・仕様は変わりやすいため、最新情報は必ずAnthropic公式でご確認ください。

**この記事でわかること**

- 新モデル「Claude Sonnet 5」の発表内容と特徴
- Claude Codeで既定モデルが切り替わったこと（バージョンと時期）
- 料金と、Opus 4.8 / Haiku 4.5との現実的な使い分け

## Claude Sonnet 5とは

Anthropicは2026年6月30日、新モデル「Claude Sonnet 5」を発表しました。公式によると「これまでで最もエージェント的なSonnetモデル」と位置づけられています。

具体的には、計画を立て、ブラウザやターミナルといったツールを使い、自律的にタスクを進める能力が強化されたとされています。推論・ツール利用・コーディング・知識労働の各面で改善したと説明されています。

技術面では、コンテキストウィンドウ（一度に扱える文章量）は100万トークンに対応します。これは大規模なコードベースや長い会話を一度に読み込めることを意味します。

**Claude Codeユーザーへの影響**: 大きなリポジトリを丸ごと読ませたり、長時間のエージェント作業をさせたりする用途で扱いやすくなりました。Sonnet系は元々「速さとコストのバランス型」で、その世代が刷新された形です。

出典: Anthropic公式発表（末尾リンク参照）

## Claude Codeで既定モデルに切り替わった

読者にとって最も直接的な変化はここです。Claude Code側の更新情報によると、**バージョン2.1.197（2026年7月1日）でClaude Sonnet 5がClaude Codeの既定（デフォルト）モデルになった**と案内されています。1Mトークンのコンテキストと、後述の割引料金付きです。

なお、Anthropicの発表では「無料プランとProプランの既定モデル」とされています。Max・Team・Enterpriseなど上位プランでの既定の扱いは、利用しているプランや設定によって異なる場合があります。自分の環境での既定は、Claude Codeの`/model`コマンドや設定画面で確認するのが確実です。

**Claude Codeユーザーへの影響**: 特に何も設定を変えていない場合、次にClaude Codeを更新して使うと、応答するモデルがSonnet 5に変わっている可能性があります。「いつもと挙動が違う」と感じたら、まずモデルが切り替わっていないかを疑うとよいでしょう（筆者の見方）。従来どおりのモデルを使いたい場合は`/model`で明示的に指定できます。

## 料金（2026年7月3日時点）

Claude Sonnet 5のAPI料金は次のとおりです。

- 標準料金: 入力100万トークンあたり3ドル / 出力100万トークンあたり15ドル
- 導入割引: 入力100万トークンあたり2ドル / 出力100万トークンあたり10ドル（**2026年8月31日まで**）

つまり8月末までは割引価格で試せます。9月以降は標準料金に戻る案内が出ています。

サブスクリプション（Pro / Max等）で使う場合は、トークン単価ではなくプランの利用上限の中で使う形になります。最新の料金・上限は変わりやすいため、公式の料金ページで確認してください。

**Claude Codeユーザーへの影響**: 後述のOpus 4.8（入力5ドル/出力25ドル）と比べると、Sonnet 5は割引期間中で半分以下の単価です。API従量課金でコストを気にしながら大量に回す使い方では、既定がSonnet 5になったことは実質的な値下げに近い効果があります。

## 既存モデルとの使い分け

Sonnet 5がすべてを置き換えるわけではありません。現行の主なモデルの位置づけを、料金の目安とともに整理します（料金は100万トークンあたり）。

- **Claude Sonnet 5**: 割引中は入力2ドル/出力10ドル（標準は3ドル/15ドル）。1Mコンテキスト。バランス型で、既定モデル。日常のコーディングやエージェント作業の主力候補
- **Claude Opus 4.8**（2026年5月28日リリース）: 入力5ドル/出力25ドル。特に難しい設計・複雑なタスク向けの最上位クラス
- **Claude Haiku 4.5**: 入力1ドル/出力5ドル。小型・高速・低価格。定型的な修正や軽いタスク向き
- **Claude Fable 5**: 入力10ドル/出力50ドル。最高性能だが高価。難所限定で投入する使い方が現実的

使い分けの目安は次のとおりです（筆者の見方を含みます）。

- 日常のコーディング全般: まずは既定のSonnet 5で十分なことが多い
- 特に難しい設計・詰まったとき: Opus 4.8やFable 5に切り替え
- 単純作業・コスト最優先: Haiku 4.5
- コストを抑えつつ量をこなす: 割引中のSonnet 5が有力

「難所だけ上位モデルに切り替え、普段は安いモデルで回す」のが、コスト効率の良い基本方針と考えられます。

## まとめ

- 2026年6月30日にClaude Sonnet 5が登場。最もエージェント的なSonnetとされ、1Mコンテキストに対応
- Claude Codeではv2.1.197（7月1日）で既定モデルに。無料・Proプランの既定でもある
- 割引料金は入力2ドル/出力10ドルで8月31日まで。9月以降は3ドル/15ドルの標準料金へ
- 普段はSonnet 5、難所はOpus 4.8/Fable 5、軽作業はHaiku 4.5という使い分けが現実的

状況は動いています。料金・仕様・既定モデルの扱いは、最新情報を公式でご確認ください。
