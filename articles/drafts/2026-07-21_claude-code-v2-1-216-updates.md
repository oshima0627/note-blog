# Claude Code v2.1.216の更新：長時間セッションの遅延を解消

※この記事は2026年7月20日時点の情報です。料金・仕様は変わりやすいため、最新は公式で確認してください。

## この記事でわかること

- 長時間セッションで起きていた「数秒の待ち」が解消された話
- ファイル隔離をオフにできる新設定と、その使いどころ
- `/verify`と`/code-review`が自動実行されなくなった変更

---

## 概要：v2.1.215〜v2.1.216が公開

Anthropicは2026年7月19日〜20日に、Claude Code（CLI版）のv2.1.215とv2.1.216を公開しました。前回まとめ（v2.1.212〜2.1.214）の続きにあたる更新です。

今回は新機能の追加よりも、**日常の使い勝手に効く修正**が中心です。特に長時間セッションの性能改善と、権限まわりの調整が目立ちます。以下、読者（Claude Codeを日常的に使うエンジニア）への影響が大きい順に整理します。

出典は公式changelog（[code.claude.com](https://code.claude.com/docs/en/changelog)）と、GitHubの[CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)で確認しています。

---

## 1. 長時間セッションの「数秒の待ち」を解消（v2.1.216）

v2.1.216で最も影響が大きいのが、**長いセッションで発生していた遅延の修正**です。

公式changelogでは「メッセージの正規化コストがターン数に応じて増大し、数秒の停止を引き起こしていた」問題を修正した、とされています。会話のやり取り（ターン）が増えるほど処理が重くなる性質だったため、長く使うほど体感が悪化していました。

### 読者への影響

1日中同じセッションを開いたまま作業する人ほど恩恵が大きい修正です。会話が長くなってから返答前に固まる、といった症状に心当たりがあれば、更新後は改善が期待できます。

---

## 2. ファイル隔離だけをオフにできる新設定（v2.1.216）

v2.1.216では、`sandbox.filesystem.disabled`という設定が追加されました。これは**ファイルシステムの隔離（サンドボックス）をスキップしつつ、ネットワークの外部通信制御は維持する**ための設定です。

Claude Codeのサンドボックスは、ツール実行時にファイル操作や通信を制限して安全性を高める仕組みです。今回の設定を使うと、ファイル隔離だけを外し、通信の制御は残せます。

### 読者への影響

サンドボックスのファイル隔離が既存のビルド手順やツールと相性が悪い場合の逃げ道になります。ただしファイル操作の保護を外す設定のため、**通信制御を残すとはいえ扱いには注意が必要**です。用途を理解したうえで使うのが前提です。

---

## 3. 権限・自動モードまわりの修正（v2.1.216）

権限（パーミッション）と自動モード（auto mode）に関する修正も複数入りました。公式changelogから、影響がわかりやすいものを挙げます。

- OAuthトークンの期限切れ後、自動モードが「HTTP 401」のエラーでコマンドを誤って拒否していた問題を修正
- ワークツリーで隔離したサブエージェントが、共有チェックアウトへgitを向けてしまう問題を修正
- バックグラウンドで再開したエージェントが既定エージェントに戻ってしまう問題を修正（エージェントのプロンプトとツール制限が復元されるように）
- `git`・`gh`コマンドの検証（バリデーション）を改善
- Webでの利用時、アイドル後に質問を再度聞き直す問題を修正

### 読者への影響

バックグラウンド実行やワークツリーを使った並列作業をしている人ほど、細かな不整合が減ります。特にトークン期限切れで作業が止まっていた場合は、更新で回避しやすくなります。

---

## 4. `/verify`と`/code-review`が自動実行されなくなった（v2.1.215）

v2.1.215は変更が1点だけの小さな更新です。内容は、**Claudeが`/verify`と`/code-review`のスキルを自動では実行しなくなった**というものです。

これまではセッション中に自動で走ることがありましたが、今後は必要なときに`/verify`または`/code-review`と明示的に呼び出す形になります。

### 読者への影響

意図しないタイミングで検証やレビューが走ることがなくなります。使いたいときは自分で呼ぶ運用に変わるため、**レビューを習慣にしていた人は呼び出しを忘れないよう注意**しましょう。

---

## まとめ：長く使う人ほど恩恵が大きい更新

今回のv2.1.215〜v2.1.216は、派手な新機能はないものの、長時間セッションの遅延解消・権限まわりの安定化・自動実行の見直しと、**毎日使う人ほど体感が変わる修正**がそろっています。

changelogは頻繁に更新されます。細かな挙動や設定名は変わることがあるため、実際に設定を変える際は必ず公式ドキュメントで最新を確認してください。
