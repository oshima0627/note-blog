## Claude Cowork が web・モバイルに対応

2026年7月7日、Anthropic は **Claude Cowork** を、これまでのデスクトップアプリに加えて **web とモバイルでも利用できるようにした**と発表しました（公式リリースノート）。

Claude Cowork は、複数ステップの作業をエージェントに任せて進めてもらうための製品です。デスクトップ版は2026年1月に登場しており、今回のアップデートで対応プラットフォームが広がった形です。

公式リリースノートでは「Claude Cowork is now available on web and mobile in addition to desktop（デスクトップに加え web とモバイルで利用可能になった）」と説明されています。提供は **Max プランから順次**始まり、その後に他プランへ広げていくとされています。

**読者への影響**：端末に縛られずにエージェント作業を扱えるようになります。デスクトップで始めた作業を、外出先ではスマホで状況を確認し、あとで結果を受け取る、といった使い方が想定されています。

## 「閉じても動く」リモートセッションとスケジュール実行

今回の対応で中心となるのが、**セッションをリモート（クラウド側）で実行する仕組み**です（現時点ではベータ）。

公式の説明によると、セッションとファイルは Claude アカウントに保存され、どの端末からでも引き継げます。そして「Work continues when you close your laptop, and scheduled tasks run with no device online（ノートPCを閉じても作業は続き、スケジュールされたタスクは端末がオフラインでも実行される）」とされています。

つまり、ローカル端末を起動し続けていなくても、あらかじめ組んだタスクが決まった時刻に自動で動くということです。TechCrunch の報道では、Anthropic が示した例として「月曜の朝6時にクライアント向けの準備を仕込んでおくと、Claude がメールスレッドや議事録、最新情報を読み込んでブリーフィング資料を作り、フォローアップのメールを下書き（未送信）まで用意しておく」というワークフローが紹介されています。

**読者への影響**：Claude Code の周辺でも、すでに「クラウド上でエージェントを動かす」流れ（Claude Code on the web やルーチンの定期実行など）が進んでいました。今回の Cowork の web・モバイル対応は、その「端末に依存せず、閉じても動く」方向性を、開発以外の日常業務にも広げるものと位置づけられます。

なお Anthropic は、匿名化した約120万件の Cowork セッション（60万以上の組織、2026年5月下旬時点）の分析も公開しました。用途の内訳はビジネスプロセス業務が33.4%、コンテンツ作成が16.4%で、ソフトウェア開発は8.7%だったとされています。開発用途に限らず幅広い業務で使われている状況がうかがえます。

## 同日発表：Microsoft 365 の書き込みツール

同じ7月7日には、**Claude の Microsoft 365 連携に書き込みツールが追加された**ことも発表されました。

公式リリースノートによると、書き込みツールを有効にすると、Claude はメールの下書き・送信・整理、カレンダー予定の管理、メールボックス設定の更新、OneDrive や SharePoint のファイル作成・更新ができるようになります。読み取り・検索ツールは従来どおりで、Teams は引き続き読み取り専用とされています。

**読者への影響**：業務で Microsoft 365 を使う開発者にとっては、資料作成やメール処理といった「開発の前後の作業」を Claude 側に寄せられる余地が広がります。ただし送信・更新まで行える機能のため、権限や運用ルールの確認は必要です。

## 筆者の見方

今回の一連の発表は、Anthropic が「エージェントを端末から切り離し、クラウドで動かし続ける」方向に軸足を移していることを示していると考えられます（筆者の見方）。

Claude Code を CLI やエディタで使っているエンジニアにとっては、直接コードを書く体験そのものが変わるわけではありません。ただ、定期実行・リモート実行を前提に「作業を仕込んでおく」設計を、業務全体で意識する価値は高まりそうです。提供範囲はプランごとに段階的なので、自分の環境でどこまで使えるかは公式で確認するのが確実です。
