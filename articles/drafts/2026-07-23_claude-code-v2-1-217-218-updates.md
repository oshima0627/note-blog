# Claude Code v2.1.217/218の更新：サブエージェントの同時実行に上限

※この記事は2026年7月22日時点の情報です。仕様は変わりやすいため、最新は公式で確認してください。

## この記事でわかること

- サブエージェントの「同時実行数」と「入れ子生成」に上限が付いた話
- `/code-review`と`/deep-research`が勝手に走らなくなった変更
- 毎日使う人に効く安定性・安全性の修正

---

## 概要：v2.1.217〜v2.1.218が公開

Anthropicは2026年7月21日〜22日に、Claude Code（CLI版）のv2.1.217とv2.1.218を公開しました。前回まとめ（v2.1.215〜2.1.216）の続きにあたる更新です。

今回はサブエージェント（子エージェント）の暴走を防ぐ制御の追加と、レビュー・調査系スキルの自動実行の見直しが目立ちます。読者（Claude Codeを日常的に使うエンジニア）への影響が大きい順に整理します。

出典は公式changelog（[code.claude.com](https://code.claude.com/docs/en/changelog)）で確認しています。

---

## 1. サブエージェントの同時実行数に上限（v2.1.217）

v2.1.217で最も影響が大きいのが、**サブエージェントの制御まわり**の変更です。サブエージェントとは、1つの指示から並行して起動される子作業エージェントのことです。

公式changelogでは、次の3点が挙げられています。

- 同時に走るサブエージェントの数に上限を追加。**既定は20**で、環境変数`CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS`で変更できます。1つのメッセージが際限なくバックグラウンドのエージェントを増やすのを防ぐ狙いです。
- サブエージェントが**さらに入れ子でサブエージェントを生成しない**よう既定を変更。より深い入れ子を許可したい場合は`CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH`を設定します。
- `--max-budget-usd`が**バックグラウンドのサブエージェントも止める**よう修正。上限に達すると新規の生成が拒否され、実行中のバックグラウンドエージェントも停止します。

### 読者への影響

並列エージェントを多用する人ほど関係します。これまでは指示次第でエージェントが無制限に増え、コストやマシン負荷が読みにくい面がありました。今回の上限で、**想定外の並列爆発と課金の膨張を抑えやすく**なります。意図的に多数を並列させたい場合は環境変数で上限を引き上げます。

---

## 2. `/code-review`と`/deep-research`が自動では走らない（v2.1.218）

v2.1.218では、レビュー・調査系スキルの起動方法が見直されました。

- `/code-review`を**バックグラウンドのサブエージェントとして実行**するよう変更。レビュー作業で会話が埋まらなくなります。
- `/deep-research`は**手動で呼び出したときだけ開始**するよう変更。Claudeが自分の判断で勝手に起動することはなくなりました。
- `context: fork`を持つスキルは**既定でバックグラウンド実行**に。従来どおり前面で動かしたい場合は、スキルごとに`background: false`を指定します。

### 読者への影響

前回のv2.1.215で`/verify`と`/code-review`の自動実行が止まったのに続く流れです。**意図しないタイミングで重い処理が走ることが減り**、会話の見通しも良くなります。使いたいときは自分で明示的に呼び出す運用に統一されていきます。

---

## 3. 入力補助の追加とログ喪失の警告（v2.1.217）

日常の入力まわりにも追加があります。プロンプト入力で**絵文字ショートコードの補完**に対応し、`:heart:`で❤️を挿入、`:hea`まで打つと候補が出ます（`emojiCompletionEnabled`設定でオフ可）。

あわせて、ディスク満杯などで**トランスクリプト（会話ログ）の書き込みが失敗**しているときや、環境変数の影響で保存がオフになっているときに警告を出すよう変更されました。これまで気づかないうちにログを失っていた状況を、事前に知らせます。

---

## 4. 安定性・安全性の修正（v2.1.217）

毎日使う人に効く不具合修正も複数入りました。公式changelogから、影響がわかりやすいものを挙げます。

- 切り詰めたMCPツール出力が全結果をセッション終了まで保持していた**メモリリークを修正**
- Windowsの自動更新失敗で`claude.exe`が消える問題を修正（失敗時は退避した実行ファイルを自動復元）
- バックグラウンドセッションの隔離が**シンボリックリンク先を正規化しておらず**、作業フォルダの外へ出られた問題を修正
- Claude Desktopで企業のmTLS・TLS検証・プロキシ設定が無視されていた問題を修正

長時間セッション・Windows環境・企業ネットワークで使う人ほど恩恵があります。特にバックグラウンド実行の隔離修正は安全性に関わるため、該当環境の人は更新をおすすめします。

---

## まとめ：エージェントの「暴走防止」がテーマ

今回のv2.1.217〜v2.1.218は、サブエージェントの同時実行・入れ子・予算の制御と、レビュー/調査スキルの自動実行の見直しが中心でした。**エージェントが勝手に増えたり走ったりするのを抑える**方向の更新がそろっています。

changelogは頻繁に更新されます。設定名や既定値は変わることがあるため、実際に環境変数を設定する際は必ず公式ドキュメントで最新を確認してください。
