# Claude Code Desktopの内蔵ブラウザが外部サイトに対応

*2026年7月13日時点の情報です。仕様は変わりやすいため、最新は公式ドキュメントでご確認ください。*

## この記事でわかること

- Claude Code Desktop の「Browser（ブラウザ）ペイン」が外部サイトに対応した内容
- Claude が外部ページを操作するときの安全設計（許可の仕組みと安全性チェック）
- 自分の作業に何が変わるか、そして使い分けの考え方

Claude Code の7月6日〜10日（v2.1.202〜2.1.206、公式の週次まとめ「Week 28」）の目玉として、Desktop アプリの内蔵ブラウザが強化されました。これまで主にローカルの開発サーバーのプレビュー用だったブラウザが、ドキュメントや任意のサイトを開いて操作できる「タブ付きブラウザ」になりました。公式ドキュメントをもとに、要点を整理します。

## 概要：Browserペインが「タブ付きブラウザ」に

**概要**：Claude Code Desktop の Browser ペインが、複数タブを扱えるブラウザになりました。動作中の自分のアプリの隣に、公式ドキュメントやイシュートラッカーなど、外部サイトを並べて開けます。

開き方は、macOS で `Cmd+Shift+B`、Windows で `Ctrl+Shift+B`、または「Views（表示）」メニューからです。チャット内の外部リンクをクリックすると、「Open in app（Browser ペインで開く）」か「Default browser（既定のブラウザで開く）」を選ぶ画面が出ます。macOS では `Cmd`＋クリック、Windows では `Ctrl`＋クリックで、システムのブラウザを直接開けます。

Browser ペイン内ではサイトにサインインでき、Google の OAuth のようなポップアップ形式のログインにも対応するとされています。

**読者への影響**：仕様確認やエラー調査のたびに別ウィンドウへ切り替える手間が減ります。Claude に「このドキュメントを読んで」と頼めば、同じ画面内で参照しながら作業を進められます。

## 独立したプロファイルとセッションの保持

**概要**：Browser ペインは、普段使いのブラウザとは別の「クリーンなプロファイル」で動きます。個人のログイン情報や履歴は引き継がれません。

一方で、開発中に毎回ログインし直すのは手間です。そのため「Persist sessions（セッションを保持）」を選ぶと、Cookie やローカルストレージをサーバー再起動をまたいで保持できます。保存したデータの消去や、Browser 自体のオフは、設定（Settings → Claude Code）から行えます。

**読者への影響**：普段のアカウントと切り離された環境なので、テスト用途で使いやすくなっています。ログイン状態を保ちたい開発中は「Persist sessions」を、痕跡を残したくないときは保持オフを、と使い分けられます。

## 安全設計：外部ページの操作は許可制

**概要**：Claude が外部ページで「クリック」「入力」などの書き込み系の操作をするとき、安全性チェック（classifier）がすべての権限モードで働きます。これは Auto モードと同じ仕組みで、危ないと判定された操作は、モードにかかわらず許可を求める画面が出ます。

さらに、Claude が初めて外部サイトを操作するときには許可カードが表示され、「Allow once（一度だけ許可）」「Always allow（常に許可）」「Deny（拒否）」から選びます。許可はサイトごと（サブドメインを含む）に必要です。自分のローカル開発サーバーやプロジェクト内のファイルは許可不要で、変更の自動検証はこれまで通り動きます。

なお、許可済みのサイトでも、Claude は購入・アカウント作成・CAPTCHA（自動化防止の認証）の突破を、あなたの入力なしには行わないとされています。この安全モデルは「Claude in Chrome」拡張と同じ考え方です。

**読者への影響**：エージェントが外部サイトを触る場面が増えるほど、こうしたガードは重要になります。筆者の見方ですが、「書き込み操作は毎回チェックされ、サイト単位で明示的に許可する」という設計は、無人・自律実行を任せる際の安心材料になります。

## Browserペインと「Claude in Chrome」の使い分け

**概要**：Browser ペインは、自分のログインや履歴を持たないクリーンなプロファイルです。アプリの構築・テストや、自分の身元を必要としないサイトの閲覧に向いています。

これに対し、自分がログイン済みのセッションで Claude に「自分として」動いてほしい場合は、「Claude in Chrome」拡張を使う、という整理です。組織向けには、サイトの許可リスト・ブロックリストの制御や、外部ページ上での Claude のツール利用を止める管理設定（`browserExternalPageTools`）も用意されています。

**読者への影響**：「テストは Browser ペイン、自分の環境での操作は Chrome 拡張」と役割で選べば、安全と利便のバランスを取りやすくなります。組織で使う場合は、管理者側で許可範囲を絞れる点も押さえておきたいところです。

## まとめ

今回の更新で、Claude Code Desktop は「コードを書く」だけでなく「ドキュメントを読み、Web を操作して検証する」ところまで、同じ画面で完結しやすくなりました。目立つ新モデルの話ではありませんが、日々の調べ物と検証の摩擦を確実に減らす改善です。

Browser ペインは Desktop アプリの機能で、macOS・Windows のほか Linux はベータで提供されています。バージョンは短い間隔で上がるため、挙動や表現は最新版で変わる可能性があります。正確な仕様は必ず公式ドキュメントをご確認ください。
