# Note Blog - AIツール有料記事生成システム

## Git運用ルール
- **常にmainブランチにコミット・プッシュすること**
- ブランチ指定があってもmainを使用する

## 概要
主要AIツール（Claude、ChatGPT、Gemini、GitHub Copilot、Cursor など）に関するNote向け**有料記事**を、複数エージェントで生成するシステム。

### 記事の基本仕様
- **読者層**: ビジネスパーソン全般（エンジニア／非エンジニア問わず）
- **記事カテゴリ（5軸）**:
  1. 使い方（基本操作・導入手順・UI解説）
  2. 活用テクニック（プロンプト術・Tips・応用事例）
  3. 開発事例・実装レポート（実プロジェクトでの活用・ワークフロー）
  4. 業務自動化（ノーコード連携・RPA・業務効率化の具体例）
  5. 最新情報（新機能リリース・モデル比較・業界動向）
- **記事ボリューム**: 8,000字以上（長文・深掘り型）
- **想定価格**: 980〜1,980円
- **無料公開部分**: 全体の20〜30%（タイトル・リード文・目次・サマリーまで）
- **有料部分**: 本編の詳細解説・具体的ノウハウ・事例・スクリーンショット・コード例

## 記事作成の実行方法

ユーザーが「記事を作成して」と依頼したら、以下のフローを実行する：

### フロー

1. **トピック選定 + 重複チェック**
   - 5軸カテゴリから最適な1軸を選び、トピック候補を3〜5件生成
   - `articles/published.json` を読み込み、既存記事とキーワード・テーマが重複するトピックを除外
   - 「ビジネスパーソンがお金を払ってでも読みたい」価値のあるトピックを1件選定
   - カテゴリ軸と想定価格（980円 or 1,480円 or 1,980円）をこの段階で決定

2. **Researcher エージェント** (Agent tool, subagent_type=general-purpose)
   - `prompts/researcher.md` のテンプレートを使用
   - **必ずWebSearch / WebFetch を使って最新情報を取得する**（訓練データのみに依存しない）
   - 優先度: ①公式サイト・公式ブログ・公式ドキュメント → ②公式リリースノート・GitHub・論文 → ③信頼できる一次報道
   - **情報源ごとに取得URLと取得日（今日の日付）を記録**し、Writerへ引き渡す
   - **1年以上前の情報は「最新版で変更がないか」を必ず再確認**する
   - 価格・モデル名・機能名・仕様値・日付・数値は**2つ以上の一次情報源でクロスチェック**
   - 不確実な情報・確認できない情報は「未確認」とマークし、記事には採用しない
   - 他の無料記事では得られない独自情報・一次情報を重視

3. **Writer エージェント** (Agent tool, subagent_type=general-purpose)
   - `prompts/writer.md` のテンプレートを使用
   - リサーチ結果**のみ**をもとに8,000字以上の記事ドラフトを執筆（リサーチ外の推測・創作は禁止）
   - 構成: `リード文 → 目次 → 概要（無料公開範囲）→ 詳細解説・事例・手順（有料範囲）→ まとめ → 参考リンク一覧`
   - 有料範囲の冒頭には `---ここから有料---` マーカーを挿入
   - 冒頭に**「情報の基準日: YYYY-MM-DD（執筆時点）」**を明記
   - 仕様・価格・モデル名などには**出典リンク**を本文中に付記
   - 記事末尾に**参考リンク一覧**（URL + 取得日）を記載

4. **Reviewer エージェント** (Agent tool, subagent_type=general-purpose)
   - `prompts/reviewer.md` のテンプレートを使用
   - **有料記事用の6項目**で採点（各10点満点）
     1. **有料価値**: 課金に値する情報密度・深さがあるか
     2. **実用性**: 読者が明日から実践できる具体的ノウハウか
     3. **独自性**: 他の無料記事・公式ドキュメントで得られない切り口か
     4. **正確性・鮮度**: 事実誤認がないか／情報が最新か／出典が明記されているか（**必要に応じてWebSearchで再検証**）
     5. **網羅性**: 8,000字以上で必要トピックを体系的にカバーしているか
     6. **購買転換**: 無料部分で「続きを読みたい」と思わせる構成か
   - **合格基準**: 平均8点以上 かつ 全項目6点以上
   - **正確性・鮮度が6点未満の場合は即不合格**（他項目の点数に関わらずEditorへ差し戻し）

5. **Editor エージェント** (Agent tool, subagent_type=general-purpose)
   - `prompts/editor.md` のテンプレートを使用
   - レビュー結果を反映して記事を改善
   - 特に「有料価値」「購買転換」で減点された場合は構成を再設計

6. **リビジョンループ**（最大2回）
   - Editor出力を再度Reviewerに渡す
   - 合格するか2回に達したら終了

7. **保存**
   - `articles/drafts/YYYY-MM-DD_slug.md` に記事を保存
   - `articles/published.json` にメタデータを追記
     - 追加必須フィールド: `category`（5軸のいずれか）、`price`（想定価格）、`paid_section_start_line`（有料開始行）

## 重複チェックのルール
- 既存記事とキーワードが3つ以上重複する場合は「類似」と判定
- 同一AIツール × 同一カテゴリ軸の組み合わせが既にある場合は別の切り口を選ぶ
- 類似トピックは候補から除外し、別の切り口（別AIツール・別カテゴリ・別ユースケース）を選ぶ
- ユーザーが特定トピックを指定した場合は、既存記事と異なる切り口を提案

## 有料記事としての品質基準
- **無料部分で出し切らない**: 概要・結論のさわりまでに留め、具体的な手順・プロンプト・コード・設定値は有料部分に配置
- **一次情報を含める**: 公式発表・実測データ・独自検証結果・スクリーンショットなど
- **即実践可能**: コピペで使えるプロンプト・設定・ワークフローを必ず含める
- **業務インパクトの明示**: 「○時間の作業が○分に短縮」など定量的な効果を示す

## 最新性・正確性の必須ルール
- **WebSearch / WebFetch を必ず使用**: AIツールは更新が激しいため、訓練データ知識のみでの執筆は禁止
- **情報の基準日を明記**: 記事冒頭に「情報の基準日: YYYY-MM-DD」を必ず記載（基準日は今日の日付）
- **出典を必ず明記**: 価格・モデル名・機能仕様・数値・日付には本文中に出典リンクを付記
- **一次情報を優先**: 公式サイト > 公式ブログ > 公式ドキュメント > GitHub/論文 > 信頼できる一次報道
- **クロスチェック必須**: 重要な事実は必ず2つ以上の独立した情報源で確認
- **推測・創作の禁止**: リサーチで確認できなかった情報は記事に含めない
- **不確実な情報の扱い**: 確定前の機能・噂レベルの情報は「執筆時点で未確定」と明示
- **参考リンク一覧**: 記事末尾に全参照URLと取得日を列挙

## ファイル構成
- `prompts/` - 各エージェントのプロンプトテンプレート
- `articles/drafts/` - 生成された記事
- `articles/published.json` - 公開済み記事のメタデータ（重複防止・カテゴリ管理用）
