## 指示

以下の観点で情報を収集・整理してください:

1. **トピックの概要**: 何についての話か、なぜ今重要か
2. **技術的／業務的なポイント**: 具体的な機能・使い方・仕組み・業務影響
3. **実践的なユースケース**: 読者がすぐ試せる具体例を3つ以上（プロンプト例・設定例・ワークフロー例を含む）
4. **他ツールとの比較・優位性**: 類似ツールとの違い（該当する場合）
5. **注意点・制限事項**: 知っておくべき制約・ベストプラクティス
6. **定量的効果**: 実測値・ベンチマーク・「○時間→○分」などの業務インパクト（取得できた範囲で）

## 情報源の収集ルール

- **必ず WebSearch / WebFetch を使って最新情報を取得する**（訓練データのみに依存しない）
- 優先度:
  1. 公式サイト・公式ブログ・公式ドキュメント・公式リリースノート
  2. GitHub（公式リポジトリ）・論文
  3. 信頼できる一次報道
- **情報源ごとに `url` と `accessed_on`（取得日。基準日 {asof_date} を使用）を必ず記録**
- **1年以上前の情報は「最新版で変更がないか」を必ず再確認**する
- 価格・モデル名・機能名・仕様値・日付・数値は **2つ以上の独立した一次情報源でクロスチェック**
- 不確実な情報・確認できない情報は `unverified` に記録し、記事には採用しない

## 出力フォーマット

以下の構造でJSON形式で出力してください:

```json
{
  "topic": "トピック名",
  "category": "5軸カテゴリ名",
  "asof_date": "YYYY-MM-DD",
  "summary": "100文字以内の概要",
  "key_points": ["ポイント1", "ポイント2", "..."],
  "use_cases": [
    {"title": "ユースケース名", "description": "具体的な説明", "example": "コード例・プロンプト例・手順", "sources": ["url1", "url2"]}
  ],
  "comparisons": "比較情報（該当する場合）",
  "caveats": ["注意点1", "注意点2"],
  "quantitative_impact": ["○時間→○分 など具体的な効果"],
  "sources": [
    {"url": "https://...", "accessed_on": "YYYY-MM-DD", "type": "official/github/paper/news", "claim": "この情報源から得た主要事実"}
  ],
  "unverified": [
    {"claim": "確認できなかった情報", "reason": "根拠不足 / 情報源なし"}
  ],
  "recommended_structure": ["推奨する見出し構成"]
}
```

## 注意事項
- 2つ以上の一次情報源でクロスチェックできない事実は `unverified` に入れる
- ビジネスパーソン（エンジニア／非エンジニア両方）を読者として意識する
- Claude/Claude Code に限らず、割り当てられた対象AIツール全般に対応する
